# Flutter MiMo 语音 App 设计

日期：2026-05-28
状态：功能范围已确认，等待实现计划

## 目标

做一款 Android 优先的 Flutter App，用小米 MiMo AI 实现语音生成、声音克隆和音色设计。第一版要像一个实用的移动语音工作台：用户能创建或选择音色，输入文本，生成音频，播放结果，并管理历史记录。

## 产品范围

第一版包含：

- 使用 MiMo 生成文本语音。
- App 内置默认音色。
- 用户通过实时录音或上传音频克隆声音。
- 用户通过描述设计 AI 音色。
- 将设计出来的音色保存为稳定的参考音频。
- 后续使用设计音色时，统一走声音克隆生成路径。
- 管理默认音色、克隆音色和设计音色。
- 管理生成历史，支持播放、导出/分享和删除。
- 设置 MiMo 服务配置或后端代理地址。

第一版不包含：

- 社区音色分享。
- 支付、会员或额度计费 UI。
- 复杂多账号体系。
- 离线语音合成。

## 推荐架构

生产版本使用 Flutter App + 后端代理。App 负责用户体验和本地状态，后端负责保存 MiMo 凭证、适配 MiMo API、处理音频格式。

如果只是个人原型，可以临时允许用户在设置页填写 MiMo API Key，并用安全本地存储保存。这个模式必须保持可替换，后续能平滑切换到后端代理。

```text
Flutter App
  -> 录音/上传音频
  -> 管理音色和文本
  -> 调用后端，或在原型阶段临时直连 MiMo
  -> 播放、保存、导出生成音频

后端代理
  -> 保存 MiMo API Key
  -> 规范化音频格式
  -> 调用 MiMo TTS、VoiceClone、VoiceDesign
  -> 返回生成音频和元数据
```

## 核心数据模型

### 音色

即使 MiMo 不提供可复用的远程 `voice_id`，App 也要给每个保存的音色分配一个本地身份。

```text
id: 本地 UUID
name: 用户可见名称
type: builtin | cloned | designed
referenceAudioPath: 克隆音色/设计音色的本地参考音频路径
previewAudioPath: 试听音频路径
stylePrompt: 设计音色时使用的描述
createdAt: 创建时间
updatedAt: 更新时间
```

### 生成音频

```text
id: 本地 UUID
text: 生成文本
voiceId: 使用的本地音色 id
voiceName: 生成时的音色名称
audioPath: 本地生成音频路径
durationMs: 音频时长
createdAt: 创建时间
```

## 音色创建流程

### 声音克隆

声音克隆支持两种输入：

- 在 App 内实时录音。
- 上传已有 `mp3` 或 `wav` 文件。

流程：

```text
授权确认
  -> 选择实时录音或上传文件
  -> 校验音频格式和时长
  -> 用 MiMo VoiceClone 生成试听/测试语音
  -> 用户试听
  -> 保存为 AI 音色
```

保存后的克隆音色使用原始录音或上传文件作为 `referenceAudioPath`。

### 音色设计

MiMo VoiceDesign 不生成稳定可复用的远程 ID，所以设计音色要转成“本地保存的稳定参考音色”。

流程：

```text
用户输入音色描述
  -> App 使用一段固定试听文本
  -> MiMo VoiceDesign 生成试听音频
  -> 用户试听
  -> 用户保存该设计音色
  -> App 将试听音频保存为 referenceAudioPath
```

保存后，设计音色不再用 VoiceDesign 生成正式语音，而是和克隆音色走同一条路径：

```text
读取设计音色的参考音频
  -> 将参考音频传给 MiMo VoiceClone
  -> 生成用户输入的目标文本语音
```

这样可以让设计音色在多次生成、长文本拆分生成时保持更稳定的音色一致性。

## 主生成流程

用户在“生成”页完成日常语音生成。

```text
选择音色
  -> 输入文本
  -> 调整语速/情绪/风格
  -> 点击生成
  -> 展示进度
  -> 播放结果
  -> 自动保存到历史记录
```

音色路由规则：

- 默认音色使用 MiMo 内置 TTS 音色参数。
- 克隆音色使用 MiMo VoiceClone + 保存的参考音频。
- 设计音色也使用 MiMo VoiceClone + 保存的设计试听音频。

## 页面导航

使用底部导航，包含四个主 Tab：

```text
生成 | 音色库 | 历史 | 设置
```

### 生成页

日常使用的主工作台。

布局：

```text
当前音色卡片
文本输入区
语音参数：语速、情绪、风格提示
生成按钮
生成进度
音频播放器和保存/分享操作
```

### 音色库页

音色库包含两个一级 Tab：

```text
默认音色 | AI 音色
```

AI 音色包含克隆音色和设计音色。每个音色卡片支持：

- 试听。
- 使用该音色。
- 用户创建的音色可重命名。
- 用户创建的音色可删除。

主创建入口打开音色创建面板：

```text
克隆音色 | 设计音色
```

### 历史页

展示历史生成音频。

每条记录包含：

- 音色名称。
- 文本预览。
- 创建时间。
- 音频时长。
- 播放。
- 导出/分享。
- 删除。

### 设置页

设置包含：

- MiMo 服务模式：后端代理或原型直连 API Key。
- 后端地址或 MiMo API Key。
- 存储和缓存占用。
- 清空生成历史。
- 授权和隐私说明。

## 错误处理

App 需要用用户能理解的语言解释错误：

- MiMo 配置缺失：提示用户去设置页配置服务。
- 音频格式不支持：提示使用 `mp3` 或 `wav`。
- 音频太短或噪声过大：提示重新录音或上传更清晰样本。
- 生成超时：保留当前文本，允许重试。
- 网络失败：保留草稿，显示重试入口。

## 隐私和授权

声音克隆和音色设计涉及敏感的身份类数据。App 在录音、上传、克隆、保存用户创建音色前必须展示授权确认。

第一版需要明确说明：用户只能克隆自己的声音，或已获得授权的声音。

## 测试策略

初始测试覆盖：

- Flutter 页面导航和主流程 smoke test。
- 本地音色类型路由单元测试。
- 设计音色保存后必须走克隆生成路径的单元测试。
- 音频格式接受和拒绝测试。
- 在 Android 模拟器和真实 Android 设备上手动测试录音权限。
