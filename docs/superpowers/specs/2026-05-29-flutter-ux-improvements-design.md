# Flutter UX Improvements Design

## Goal

完善现有 Flutter 版 MiMo 语音 APP 的 7 个用户体验功能，让用户可以稳定保存数据、顺畅生成语音、管理音色和历史，并在 API 配置、录音克隆、文档生成等关键流程里得到清楚反馈。

## Approved Scope

本次只完善以下 7 类功能：

1. 数据持久化
2. 生成流程增强
3. 录音克隆体验
4. 音色库体验
5. 长文本和文档体验
6. 历史记录体验
7. 设置页可靠性

不做登录、会员、云同步、后台服务、支付、在线素材市场，也不更换 Flutter 技术栈。

## Architecture

继续沿用当前 `AppState + services + ui` 的结构。新增本地持久化服务负责保存用户音色、生成历史、草稿和生成参数；业务状态仍由 `AppState` 统一暴露给页面。UI 层不直接读写文件系统，全部通过服务接口调用，方便写测试。

主要边界：

- `AppState`：聚合业务状态，暴露选择音色、生成语音、保存音色、删除历史等用户动作。
- `LocalVoiceStore`：从只提供内置音色扩展为“内置音色 + 用户音色持久化”。
- 新增 `LocalHistoryStore`：保存、读取、删除、清空生成历史。
- 新增 `LocalDraftStore`：保存草稿文本、风格提示、语速、情绪、选中音色。
- `MimoService`：继续负责 MiMo API 调用，增加连接测试入口。
- UI 页面：保留当前四个主页面，分别增强交互状态，不新增复杂导航。

## Feature Design

### 1. 数据持久化

用户创建的音色、生成历史、生成草稿、风格提示、语速、情绪和上次选择的音色都要在 APP 重启后恢复。

实现策略：

- 使用 `shared_preferences` 保存结构化 JSON 元数据。
- API Key 继续用 `flutter_secure_storage`。
- 音频文件继续保存在应用文档目录，历史只保存路径和元数据。
- 启动时异步加载本地数据，加载完成后通知页面刷新。

体验规则：

- 如果本地保存的音频文件已不存在，历史记录仍显示，但标记为“文件缺失”，播放/分享按钮禁用。
- 如果上次选择的用户音色不存在，回退到第一个默认音色。
- 删除用户音色时，不自动删除历史记录；历史记录保留生成时的音色名称。

### 2. 生成流程增强

生成页要减少用户卡住的情况，并让生成结果成为下一步操作入口。

新增体验：

- API Key 缺失时，生成按钮触发设置引导，不直接显示技术错误。
- 生成状态拆成“准备请求 / 调用 MiMo / 保存音频 / 自动播放”。
- 生成失败后保留文本、音色、风格和参数，并提供重试。
- 生成完成后在当前页显示播放器、下载、分享、复制文本、用相同文本重新生成。
- 生成按钮在文本为空、音色为空、正在生成时给出明确状态。

实现策略：

- 新增 `GenerationUiState` 表示 idle/generating/success/error。
- `AppState.generateCurrentVoice()` 保留核心生成逻辑，UI 层负责展示步骤状态。
- 下载和分享继续使用 `AudioExportService`。

### 3. 录音克隆体验

克隆音色页面要让用户知道该读什么、录了多久、录完能不能用。

新增体验：

- 开始录音前显示跟读文案和录音要求。
- 录音时显示计时、录音中状态和轻量波形/音量动效。
- 停止录音后允许试听、重录、确认使用。
- 上传音频后立即校验格式、大小，并显示通过/失败原因。
- 保存前必须有音色名称和参考音频。

实现策略：

- 扩展 `AudioInputService`，返回录音路径和基础元数据。
- 复用 `AudioValidator` 做 mp3/wav、Base64 大小和文件存在校验。
- `VoiceCreationSheet` 维护本地录音状态，不把未确认音频写入音色库。

### 4. 音色库体验

音色库要支持快速找到、试听、使用和管理音色。

新增体验：

- 搜索音色名称和标签。
- 筛选：全部、官方、AI 音色、男声、女声、中文、英文、收藏。
- 收藏音色和最近使用音色。
- 点击“使用”后选中音色，并提示可回到生成页使用。
- 用户创建音色支持删除确认。

实现策略：

- 给 `Voice` 增加 `favorite`、`lastUsedAt` 等本地字段。
- 内置音色字段由本地 overlay 存储收藏和最近使用，不修改硬编码默认数据。
- `VoiceLibraryScreen` 从 Tab 结构调整为搜索 + 筛选 chips + 列表。

### 5. 长文本和文档体验

用户导入 Word/PDF/TXT 后，要知道文本是否过长，以及生成会发生什么。

新增体验：

- 导入后显示文件名、字数、预估音频时长。
- 文本过长时提示分段生成。
- 自动按段落或长度拆分为多个片段。
- 用户可以选择“生成全部”或“只生成当前片段”。
- 每个片段失败时可以单独重试。

实现策略：

- 新增 `TextSegmenter` 服务，按段落和字符数分段。
- `GenerateScreen` 显示片段数量和当前片段预览。
- 第一阶段不做音频合并，只把每段生成结果写入历史。

### 6. 历史记录体验

历史页要从“生成记录列表”变成可复用的音频资产库。

新增体验：

- 历史持久化。
- 支持播放、下载、分享、删除、重命名、收藏。
- 支持按日期、音色、收藏筛选。
- 支持“复制文本”和“用这段文本重新生成”。
- 删除前确认；批量清空也需要确认。

实现策略：

- 扩展 `GeneratedAudio`，增加 `title`、`favorite`、`fileMissing` 推导状态。
- `HistoryScreen` 增加搜索/筛选栏和更多操作菜单。
- 重用 `AudioPlaybackService`，继续保证同一时间只播放一个音频。

### 7. 设置页可靠性

设置页要帮助用户知道 API 配置是否真的可用。

新增体验：

- API Key 支持显示/隐藏。
- 保存后可点击“测试连接”。
- 测试结果区分：缺 Key、URL 无效、网络失败、HTTP 错误、响应格式异常。
- 保留“直连”和“代理”模式说明。
- 提醒正式发布版本建议走后端代理，避免在客户端暴露 Key。

实现策略：

- `MimoService` 新增 `testConnection()`。
- 测试连接使用短文本和当前配置发起轻量请求。
- 设置页保存配置后不自动测试，避免用户每次保存都消耗 API。

## Data Model Changes

`Voice` 新增：

- `favorite: bool`
- `lastUsedAt: DateTime?`

`GeneratedAudio` 新增：

- `title: String?`
- `favorite: bool`

新增本地存储 DTO：

- `StoredVoice`
- `StoredHistoryItem`
- `StoredDraft`

DTO 只用于本地 JSON 读写，不泄漏到 UI。

## Error Handling

- 所有用户操作失败都显示中文可理解信息。
- API 错误保留原始状态码，但不直接把大段响应体塞进 UI。
- 本地存储读取失败时回退为空数据，同时显示一次轻提示。
- 播放、分享、下载遇到文件缺失时提示用户重新生成。
- 删除操作必须确认，避免误删音色和历史。

## Testing Plan

新增和更新测试：

- `LocalVoiceStore`：保存/读取用户音色、收藏、最近使用。
- `LocalHistoryStore`：保存/读取/删除/清空历史。
- `LocalDraftStore`：保存并恢复草稿、参数、选中音色。
- `TextSegmenter`：按段落和长度分段。
- `AudioValidator`：校验录音文件存在、格式、大小。
- `AppState`：启动加载、生成后保存历史、选择音色更新最近使用。
- `GenerateScreen`：缺 API Key 引导、生成失败重试、文档分段状态。
- `VoiceLibraryScreen`：搜索、筛选、收藏、使用音色。
- `HistoryScreen`：筛选、重命名、收藏、复制文本、重新生成。
- `SettingsScreen`：保存、显示/隐藏 Key、测试连接反馈。

验证命令：

```bash
flutter analyze --no-pub
flutter test --no-pub
flutter build apk --debug --no-pub
```

## Rollout Order

1. 本地数据模型和持久化服务。
2. `AppState` 接入启动加载、保存、删除、更新。
3. 生成页体验增强。
4. 录音克隆体验增强。
5. 音色库搜索、筛选、收藏。
6. 文档分段生成。
7. 历史记录资产化。
8. 设置页测试连接。
9. 全量测试、Android debug APK 构建和模拟器检查。

## Non-Goals

- 不实现账号体系。
- 不上传用户历史到云端。
- 不做音频编辑器、剪辑、拼接和混音。
- 不承诺微信语音消息形态分享；本期仍以系统分享 MP3/WAV 文件为主。
- 不在客户端长期硬编码用户 API Key。
